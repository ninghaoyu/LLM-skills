

| 考核方面 | 核心要求 | 预期证据 |
| :---- | :---- | :---- |
| 一、通用实践要求 (Agentic AI Practice Requirements) |  |  |
| 1\. 推理 (Inference) | 展示使用AWS托管服务（如Amazon Bedrock、SageMaker JumpStart）中的大语言模型(LLMs)来构建可靠且具有成本效益的代理解决方案的专业知识。 | 使用来自Bedrock、Bedrock Marketplace或SageMaker AI / JumpStart的模型来交付代理解决方案的经验。 |
| 2\. SDKs/库/工具 (SDKs/Library/Tooling) | 展示使用AWS兼容的代理开发框架（如AWS Strands、LangGraph等）来实现代理工作负载的专业知识，以实现智能规划、推理和自主行动。 | 使用此类框架实施代理工作负载，实现可重用性、推理和自主行动，以构建可扩展客户解决方案的经验。 |
| 3\. 互操作性 (Interoperability) | 展示实施需要跨代理协调或与外部工具和服务集成的代理应用程序的专业知识。 | 实施安全代理互操作性的经验，使用协议（如MCP、A2A）和集成能力来连接代理与工具、服务和内存结构，并在基于解决方案的方法中保持灵活性。 |
| 4\. 安全 (Security) | 展示实施身份管理、安全控制和可观测性以保障AWS代理解决方案的专业知识。 | 实施AWS安全服务（IAM/IAC、Security Hub、CloudTrail）、Bedrock guardrails以及第三方集成（SSO/IdP）并具备全面监控和可观测性能力的经验。 |
| 5\. 负责任的AI (Responsible AI) | 展示实施全面的负责任AI控制，以确保可信的自主操作的专业知识。 | 实施负责任AI原则的经验，例如偏见缓解、人类反馈强化学习(RLHF)，应用安全护栏（如Amazon Bedrock Guardrails、Amazon SageMaker Clarify、Amazon SageMaker Ground Truth）以及人类自主操作。 |
| 6\. 计算 (Compute) | 展示在AWS托管计算服务上部署代理工作负载的专业知识。 | 利用AWS托管计算服务（如Bedrock AgentCore、Amazon EKS、Amazon ECS、AWS Lambda、AWS Fargate）部署代理解决方案，以实现优化的安全状况和成本效益的经验。 |
| 二、客户案例具体要求 (Agentic AI Consulting Services \- Customer Example) |  |  |
| 1\. 实践方法实施 (Practice Implementation Approach) | 展示如何分析客户需求、为推理选择合适的AWS基础模型、所使用的代理SDK/服务/工具，以及为解决特定业务挑战和技术环境约束所采用的集成方法。 | 架构图（展示AWS基础模型集成）、技术文档（详细说明模型选择理由和框架选择）、客户需求分析以及展示您咨询实践决策过程的实施方法论文档。 |
| 2\. 安全与互操作性实践实施 (Security & Interoperability Practice Implementation) | 展示如何设计和实施身份验证、授权和标准代理交互协议，同时在客户环境中平衡安全性能与操作功能需求以及互操作性约束。 | 为客户创建的文档，通过记录如何分析安全需求、制定适当的协议并验证安全操作，来展示您设计身份验证、授权和代理交互协议的方法。 |
| 3\. 负责任的AI实践实施 (Responsible AI Practice Implementation) | 展示如何基于生产使用模式和客户反馈，在设计、实施、测试和优化安全控制及其他负责任AI实践。 | 安全评估报告或负责任AI治理文档、在合作期间制定的测试计划或验证报告，以及展示您迭代改进过程的现有监控或反馈文档。 |
| 4\. 计算实践实施 (Compute Practice Implementation) | 展示如何利用AWS托管计算服务部署代理工作负载，实现优化的安全状况和成本效率，展示您为客户代理解决方案选择和实施适当计算基础设施的专业知识。 | 架构图（展示AWS托管计算服务部署，如Bedrock Agent、ECS、Lambda、Fargate）、详细说明架构、配置或优化策略的技术文档，以及展示在AWS托管计算基础设施上部署代理工作负载的实施指南。 |

**总结**：Agentic AI案例的考核是一个完整的体系，要求合作伙伴不仅具备构建代理式AI解决方案的**六大核心通用技术能力**（推理、工具链、互操作、安全、负责任AI、计算），还必须在**具体的客户案例**中证明能够将这些能力落地实施，并提供详尽的技术文档和架构图作为证据。
